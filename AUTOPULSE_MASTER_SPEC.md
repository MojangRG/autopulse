# AutoPulse

Mission:

Remove the burden of remembering, planning and making vehicle maintenance decisions.

The owner drives the car.
AutoPulse manages the maintenance.

Core principle:

The application must answer one question:

"What should I do with my vehicle right now?"

If a screen does not help answer that question directly or indirectly, it should be reconsidered.

Product vision:

AutoPulse is not:

- service journal
- maintenance schedule
- expense tracker
- automotive forum
- chatbot

Those are tools.

AutoPulse is:

A personal digital vehicle manager.

Sources of truth:

- VIN data
- STS data
- maintenance receipts
- service orders
- uploaded photos
- uploaded PDFs
- vehicle history
- mileage
- owner answers

AI principles:

- never give generic advice
- always use vehicle context
- always use maintenance history
- always explain recommendations
- ask questions only when information is insufficient

UI principles:

- premium
- calm
- confidence
- minimal visual noise

Main action:

Scan a document.

Long-term goal:

The owner should eventually stop worrying about vehicle maintenance entirely.
