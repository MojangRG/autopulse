# Motrix: принятые продуктовые решения

## Граница продукта

Motrix является автомобильным ассистентом. Дом, питомец, estate core и Sense не входят в продукт.

## Основное обещание

Motrix знает конкретный автомобиль, хранит подтвержденную историю, различает факты и неизвестные зоны, проверяет документы СТО и показывает одно разумное следующее действие.

## Источники данных

1. Документ, подтвержденный пользователем.
2. Данные внешнего автомобильного провайдера.
3. Сведения владельца.
4. Расчет Motrix.
5. Предположение AI, всегда с маркировкой.

## Запчасти и цены

Поставщик: Exist Opt API.

Целевой поток:

регламентная операция -> оригинальный каталог -> применимые артикулы -> оригинал и аналоги -> цена и срок -> корзина/заказ.

Подтвержденные документацией возможности Exist:

- Price/Catalogs;
- Price;
- Price/Analogs;
- Basket;
- Order;
- Delivery.

До подтверждения точной схемы запросов запрещено придумывать payload API или показывать пользователю непроверенную применимость.

## Автосервисы

Яндекс API Поиска по организациям используется для:

- поиска автосервисов;
- адресов и координат;
- телефонов;
- часов работы;
- открытия карточки и маршрута.

Публичный поиск организаций не создает запись на услугу.

Полноценная запись возможна через:

- кнопку онлайн-записи в карточке организации;
- API конкретной CRM/сети;
- партнерство с API онлайн-записи Яндекса, если Motrix соответствует требованиям партнера.

## Ближайшие этапы

1. Provider adapters и env-конфигурация.
2. Серверная база, авторизация и несколько автомобилей.
3. TRONK vehicleDataProvider.
4. Новая главная без псевдоточного health score.
5. Документы: смета отдельно, выполненные работы отдельно.
6. Контекстный AI-механик с инструментами.
7. Exist catalog integration.
8. Yandex service search UI.
9. Платежи, лимиты и фискализация.
